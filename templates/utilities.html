Clear stale Next Week due dates (#220)

Finds active tasks where tier = Next Week AND due_date ≤ today, then clears the due_date so the task moves off the calendar's today/past cells and lands in the Unscheduled aside.

Originally caused by the pre-#220 tier-button behavior: clicking "Next Week" on a task changed the tier but didn't clear the old due_date, so the calendar still rendered the task on the old day. The #220 hook prevents this for new punts; this utility cleans up the existing stuck data.

Trigger off-cycle backup (#154)

Dispatches the daily-backup.yml GitHub Actions workflow on-demand. The workflow does a pg_dump of production, Fernet-encrypts the output, commits it to the private backup repo, prunes anything older than 7 days, and emails a PASS/FAIL status report.

Useful before risky migrations, before/after a large data import, or any time you want a known-good point-in-time snapshot without waiting for the 07:30 UTC scheduled run. Requires the GITHUB_DISPATCH_TOKEN env var on Railway (fine-grained PAT, Actions: write scope).

Run weekly bug-pattern scan (#226)

Re-greps the working tree for 6 known-bad shapes that the per-commit gates miss: bare 1fr grids without minmax(0, …), embedded URL credentials in source, string-match-only prod tests, state-mutating GET routes, raw tier-string compares in Python, and unbalanced .type === "work" JS branches without a corresponding personal reference.

Same checks the weekly Sunday cron runs — this lets you verify CLEAN on-demand (e.g. before a release). The result renders inline below; no email is sent for UI runs.

Run monthly security-posture audit (#227)

Posture review (not vuln scanning). Four checks: PAT inventory against docs/security/pat-inventory.json (null expiry / > 90-day expiry / > 60-day idle); OAuth scope drift against docs/security/oauth-scopes.json (additions or removals both flagged); unencrypted sensitive columns in models.py (regex on column names cross- referenced against crypto.py import); threat-model freshness (CLAUDE.md > 180 days untouched).

Same checks the 1st-of-month cron runs — use after rotating a PAT, changing OAuth scopes, or adding a sensitive-named column to confirm the audit picks it up.

Run restore drill (#154.7)

Dispatches the monthly-restore-drill.yml GitHub Actions workflow on-demand. The workflow pulls the most recent encrypted backup, decrypts it, pg_restores into a scratch Postgres, asserts the scratch row counts are within ±5% of live (sanity), and emails a PASS/FAIL report.

Use after rotating BACKUP_FERNET_KEY, after upgrading the Postgres major version, or whenever you want evidence that the latest backup is actually restorable. "A backup you've never restored is a 50/50 backup." Requires the same GITHUB_DISPATCH_TOKEN env var.
